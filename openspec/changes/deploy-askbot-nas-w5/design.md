# Design: deploy-askbot-nas-w5

## Context

ask-bot を Synology DS920+(NAS)へ本番配備するにあたり、2026-07-21 の実地調査で配備先の実態が判明した。本 change はその実態に基づき、本番配備の前提条件・メモリ設計・配備規約・データ移送・ポート競合回避・試験先行フェーズを仕様化する。

## 実地調査で判明した事実(2026-07-21)

- ハードウェア/OS: Synology DS920+ / DSM 7.3.2。
- コンテナ基盤: Docker 24.0.2 + Compose v2.20.1。
- メモリ: 実測 3.7GiB + swap 4.2GiB。増設用 RAM は購入済みだが未装着。
- ストレージ: `/volume1` 空き 8.7TB。ルート FS 空きは 829MB のみ。
- ネットワーク: Tailscale 直結。
- 既存状態: 旧デプロイ物は完全撤去済み。ネイティブ PostgreSQL が `127.0.0.1:5432` を占有。

オーナー確認: RAM は購入済みだが未装着。装着完了まで NAS 本番配備は保留。挙動調査は試験環境で先行する。

## 決定と根拠

### 本番配備ゲート = RAM 増設完了(16GiB 以上)

実測 3.7GiB では ask-bot の全モジュールを安定稼働させられない。増設用 RAM は購入済みで、装着完了(16GiB 以上)を本番配備の前提ゲートとする。装着完了までは本番配備を行わず、試験環境で挙動を先行調査する。ゲートを仕様の前提条件(NAS-01)として明示することで、RAM 不足のまま本番配備が走ることを防ぐ。

### メモリ設計 — 総予算 16GiB の内側で最低保証 + 上限

各モジュールに `deploy.resources.reservations.memory`(最低保証)と `deploy.resources.limits.memory`(上限)を設定し、上限内で動的調整する。最低保証の総和は総予算 16GiB を超えない。これにより、負荷に応じて各モジュールが上限まで伸縮しつつ、最低保証の合計が予算を割らないことを保証する(NAS-02)。

### 配備規約 — パス・named volume・機密・最小権限

- 配置は `/volume1/docker/ask-bot/` 配下に集約する。ルート FS 空きが 829MB と逼迫しているため、イメージ・ボリューム・一時ファイルはすべて `/volume1`(空き 8.7TB)側に置く。
- 永続データは named volume で保持し、バインドマウントの散在を避ける。
- 機密は `.env` + `${VAR:?required}` 方式で注入する。必須変数が未設定なら起動を失敗させ、機密をイメージ/リポジトリに焼き込まない。
- コンテナは `read_only` + `cap_drop: ALL` + `no-new-privileges` の最小権限で稼働させる(NAS-03)。

### データ移送 — tar 圧縮ストリームの SSH 直送

ルート FS が逼迫しているため、移送は tar 圧縮ストリームを SSH で直送し、大きな中間一時ファイルをルート FS 上に滞留させない。Tailscale 直結のため SSH 経路は確立済み(NAS-04)。

### ポート競合回避 — 5432

ホストのネイティブ PostgreSQL が `127.0.0.1:5432` を占有しているため、ask-bot の DB ポートは別ポート/別バインドで公開して競合を避ける(NAS-05)。

### 試験環境での先行挙動調査

RAM 増設完了まで本番配備を保留する間、試験環境で ask-bot の挙動(メモリ実消費・ポート・起動順・依存)を先行調査し、その結果を本番配備の前提評価へ反映する(NAS-06)。

## リスクと対応

- **ルート FS 逼迫(829MB)**: イメージ・ボリューム・一時ファイルを `/volume1` 側に固定し、tar 圧縮ストリームの SSH 直送で中間ファイルを避ける。DSM/Docker のデータルートが `/volume1` にあることを配備前に確認する。
- **メモリ予算の粒度**: 各モジュールの `reservations` / `limits` の初期値は試験環境の実測(NAS-06)に基づいて調整する。初期見積りが粗いと OOM または予約過多になりうるため、試験先行で実消費を測ってから本番値を確定する。
- **swap 依存**: swap 4.2GiB に依存した安定化は避け、RAM 増設(NAS-01)を前提とする。増設前の本番配備はしない。
- **5432 競合の見落とし**: 別ポート/別バインドの設定漏れで起動衝突しうる。Compose の port 定義を配備前レビュー項目とする。

# pricing-model(§9.1)

## ADDED Requirements

### Requirement: 原価ベースの価格設計

システムは、N-25 の org 別原価実績とβ提供価値の段(Lv0〜Lv2)を入力として価格を設計し、原価の3倍を下回る価格を付けてはならない (MUST NOT)。

#### Scenario: 原価の3倍を下限とする
- **WHEN** ある料金プランを設定する
- **THEN** org 月次原価の3倍を下回らない

### Requirement: 課金モデルの方針

システムは、組織月額(規模帯別)+ 従量(推論・書き込み量)のハイブリッドを採用し、seat 課金を第一候補としてはならない (MUST NOT)。

#### Scenario: seat 課金を主軸にしない
- **WHEN** 課金モデルを定義する
- **THEN** seat 課金が主たる課金軸にならない(人を減らすほど安くなる誘因を持ち込まない)

# provider-launch(§9.3)

## ADDED Requirements

### Requirement: 提供開始の形態

システムは、βからの転換(2〜3組織)と上限を設けた招待制の新規受付で提供を開始しなければならない (SHALL)。

#### Scenario: 招待制で受け付ける
- **WHEN** 新規組織を受け付ける
- **THEN** 上限を設けた招待制で受け付ける

### Requirement: 提供開始の DoD

システムは、課金が発生する契約を1件以上獲得し、オンボーディングが手順書どおり1日以内に完了し、サポート窓口と SLA(ベストエフォートの明文化)を定義済みにしなければならない (SHALL)。

#### Scenario: 課金契約が1件以上ある
- **WHEN** 提供を開始する
- **THEN** 課金が発生する契約が1件以上ある

#### Scenario: オンボーディングが1日以内に完了する
- **WHEN** 新規組織をオンボーディングする
- **THEN** 手順書どおり1日以内に完了する

#### Scenario: SLA が定義済みである
- **WHEN** サポート方針を確認する
- **THEN** サポート窓口と SLA(ベストエフォート)が定義済みである

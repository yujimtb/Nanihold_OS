# Proposal: Agent Runtime と VSM 完成機能(サブスクCLI駆動化)

- 起案: 2026-07-12(オーナー決定に基づく)
- ステータス: 承認済み・実装中

## 背景と決定事項(オーナー指示の原文要約)

1. **頭脳と手は分離しない。** AIエージェントはAPI従量(青天井請求リスク)ではなく、**サブスクリプションCLI(Claude Code / Codex)経由**で VSM の中に入れる。architecture.md §2 の「Agent は LLM、Codex、Claude Code、人間を同じ抽象で扱う」の本実装にあたる。
2. **AIバックエンドは差し替え可能にする。** 特定モデルのサブスク提供終了が予告されているため、設定ファイル1箇所の変更でロールごとにバックエンド/モデルを切り替えられること。
3. **Budget システムは必須。** トークン In/Out(+キャッシュ読取)と実行時間(wall clock)の両方を記録・分配・強制する(architecture.md §14 の実装)。
4. **トークン削減機構。** 素朴な使い捨て起動ではコンテキストの読み直しが発生する。対策は2層: (a) 設計準拠の context view / TaskSummary 構築、(b) CLI セッション再開(**Run内・同一Node内に限定**。Event_Log が Source of Truth のままで、セッションは消えても再構成可能なキャッシュとして扱う)。
5. **クォータ自動復帰。** サブスクの5時間/週次リミット検知で Node を SUSPENDED に休眠させ、リセット時刻に自動 resume する。
6. **S2 調停の AI 化。** request_coordination の判断を S2 の Agent が行う。
7. **Algedonic channel の実装。** 階層をバイパスして S5 と人間に直接届く痛覚信号。Human を発信者・受信者の両方に含める。
8. **S3-S4-S5 Consortium の実装。** 合議体。基本は最上位での利用を想定するが、**抽象化してどの u-VSM 階層でも利用可能**にする。Human を参加者に含める。
9. **WebUI: 組織のライブ可視化。** どの Node で、どのモデルが、誰の指示のもと、何をやっているかがモダンな UI で一目で分かり、そこから介入(休眠・再開・停止・追加指示・合議参加)できる。
10. **指示の API 化。** WebUI だけでなく Claude Code / Codex などの外部エージェントからも HTTP API / CLI 経由で Run 投入・追加指示ができる。
11. **設定ファイルの一元化。** 上記すべての設定を `vsm.toml` に集約し、変更可能にする。

## スコープ外(本変更では扱わない)

- Run をまたぐセッション保持(architecture.md §17 のスコープ外項目と衝突するため)
- FSX 数値最適化・公共性測定・共有剰余配分(従来どおり保留)
- 通貨換算の料金表(サブスクのためトークン枠で管理。レート表は将来の後付けとする)
